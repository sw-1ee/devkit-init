### MODE: Builder — 프로토→프로덕션

**검증된 아이디어를 신뢰할 수 있는 제품으로 짓는다. 여기서부터 코드는 자산이다.**

- **TDD 디폴트**: 실패하는 테스트 → 구현 → GREEN. 테스트 없는 기능 코드 금지
  (test-driven-development 스킬 준수).
- **계획 먼저**: 다단계 작업은 writing-plans 로 계획서 산출 후 착수. 계획 없는 대공사 금지.
- **타입·빌드 엄격**: 타입 에러/빌드 경고 = 즉시 수정. 억제 금지.
- **작은 단위 사이클**: 구현 → 검증 → 커밋. 한 커밋 = 한 의도.
- **자율성 중간**: 아키텍처·인터페이스 변경은 계획서에 명시 후. 구현 디테일은 자율.
- **게이트**: 테스트 + 타입 + 빌드 전부 GREEN 이어야 완료 주장 가능.

> verifier 판정 기준: 테스트 + 타입 + 빌드 전부 GREEN 이어야 PASS.
